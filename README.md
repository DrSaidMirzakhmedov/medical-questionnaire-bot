# Medical Questionnaire Bot

Telegram-бот для структурированного сбора медицинского анамнеза пациента перед онлайн- или очной консультацией. Поддерживает русский и узбекский языки, сохраняет анкеты локально и (опционально) в Google Sheets, поддерживает приём оплаты через Telegram Payments.

Изначально разработан как рабочий инструмент для практики врача-инфекциониста; опубликован как шаблон, который можно адаптировать под любую специализацию.

> ⚠️ **Это инструмент сбора данных, а не медицинское ПО для диагностики.** Бот не даёт пациенту никаких клинических рекомендаций и не заменяет консультацию врача — он только структурированно собирает анамнез для последующего разбора специалистом.

## Возможности

- Пошаговый опрос через `ConversationHandler` — жалобы, длительность симптомов, температура, контакт с инфекционными больными, хронические заболевания, аллергии, принимаемые препараты, прививки, перинатальный анамнез (для педиатрических случаев), развитие ребёнка, питание, социальный и гинекологический анамнез.
- Интерфейс на **русском и узбекском** языке, переключаемый в начале диалога.
- Валидация вводимых данных (даты, числовые значения роста/веса и т.п.).
- Сохранение анкеты в локальный JSONL-файл (всегда, как надёжный бэкап) и опционально — в Google-таблицу через сервисный аккаунт.
- Опциональная оплата консультации через Telegram Payments (счёт выставляется после отправки анкеты; шаг полностью пропускается, если оплата не настроена).
- Устойчивая обработка ошибок (`error_handler`, `safe_edit_or_send`) и таймаут диалога (30 минут неактивности сбрасывает сессию).

## Технологии

## Установка

```bash

cd medical-questionnaire-bot
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Linux/macOS
pip install -r requirements.txt
```

## Настройка

1. Скопируйте `.env.example` в `.env`:
   ```bash
   copy .env.example .env        # Windows
   # cp .env.example .env        # Linux/macOS
   ```
2. Получите токен бота у [@BotFather](https://t.me/BotFather) (`/newbot`) и укажите его в `.env` как `BOT_TOKEN`.
3. Запустите бота:
   ```bash
   python bot.py
   ```

Без дополнительной настройки бот полностью работает: собирает анкеты и пишет их в `responses.jsonl`. Оплата и Google Sheets — независимые опциональные модули, включаются отдельно.

### Оплата через Telegram Payments (опционально)

1. В @BotFather: **Payments → Connect a provider**, выбрать одного из доступных платёжных провайдеров для вашего региона.
2. Указать в `.env`:
   ```
   PROVIDER_TOKEN=токен_от_провайдера
   PAYMENT_CURRENCY=UZS
   PAYMENT_AMOUNT=500000
   ```
   `PAYMENT_AMOUNT` указывается в минимальных единицах валюты — уточните множитель для вашей валюты и провайдера в [документации Bot API Payments](https://core.telegram.org/bots/payments).
3. Если `PROVIDER_TOKEN` не задан или `PAYMENT_AMOUNT=0` — шаг оплаты автоматически пропускается, бот работает как обычно.

### Google Sheets (опционально)

1. В [Google Cloud Console](https://console.cloud.google.com/) создать проект, включить **Google Sheets API**, создать сервисный аккаунт и скачать JSON-ключ.
2. Сохранить ключ как `credentials.json` в корне проекта (файл в `.gitignore`, в репозиторий не попадёт).
3. Расшарить целевую Google-таблицу на email сервисного аккаунта (поле `client_email` в JSON-ключе) с правом **Редактор**.
4. Указать в `.env`:
   ```
   GOOGLE_SHEET_ID=id_таблицы_из_ссылки
   GOOGLE_SERVICE_ACCOUNT_FILE=credentials.json
   ```

## Данные и конфиденциальность

Бот собирает медицинские данные пациентов. При адаптации под реальное использование:

- Не публикуйте `responses.jsonl` и `credentials.json` — они уже в `.gitignore`, но проверяйте это перед каждым коммитом.
- Убедитесь, что хранение и обработка собираемых данных соответствуют применимому в вашей стране законодательству о персональных и медицинских данных.
- `.env` с реальными токенами никогда не должен попадать в git.
